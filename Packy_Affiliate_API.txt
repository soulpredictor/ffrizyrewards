
Affiliate Leaderboard & Stats API
Documentation for Creator Partners
Overview
This API provides creator partners with two endpoints to access affiliate performance data for users referred through their codes.
• Stats endpoint - a flexible, date-ranged top-30 leaderboard of users who used any of your referral codes, sorted by amount wagered (USD).
• Leaderboard endpoint - your last 10 approved leaderboard events (upcoming, active, and ended) with prize tiers and the live top-30 entries for each.
Both endpoints use the same API key authentication and are intended for external use (dashboards, widgets, embeds, etc.).
Authentication
Every request must include your unique API key in the request header. Your API key will be provided to you separately - do not share it with anyone.
Header name:
X-API-Key: your-api-key-here
The base URL will be shared with you separately along with your API key. All examples below use {BASE_URL} as a placeholder.
Endpoint 1 - Affiliate Stats
GET  /v1/affiliate/stats
Returns the top 30 users who used any of your referral codes, sorted by total amount wagered (USD) within the requested date range. Useful for arbitrary reporting periods (daily, weekly, monthly, custom).
Query Parameters
All parameters are required and must be included in the URL.

Endpoint 2 - Leaderboards
GET  /v1/affiliate/leaderboard/external
Returns up to 10 of your approved leaderboards (most recently created first), including upcoming, active, and ended events. Each leaderboard includes its prize tier configuration and the top-30 live entries. Designed for external widgets and embeds - gives you both the running event and recent recaps in one call.
Entries are refreshed by a background job at most once every 5 minutes. Each leaderboard carries a next_update_at timestamp so clients can render an accurate countdown without polling aggressively.
Query Parameters
None.

When last_refreshed_at is null the entries array will be empty for that leaderboard - the event was just approved or the cache was flushed. The next refresh tick (next_update_at) will populate it; clients should display a "preparing standings" state. For ended events with no traffic this is also the expected state.

Important Notes
• Security: Keep your API key confidential. Do not share it publicly or commit it to source control.
• Caching (Stats): Stats results are cached for up to 10 minutes. Repeated requests with the same date range within this window may return identical data.
• Caching (Leaderboard entries): Entries are refreshed by a background job at most every 5 minutes. Use next_update_at to drive your countdown - there is no benefit to polling more frequently than this.
• Limits: Stats returns at most 30 users. The leaderboard endpoint returns at most 10 leaderboards, each with at most 30 entries.
• Currency: All monetary values are returned as strings in USD (e.g. "15000.50") to preserve precision.
• Date format: Dates must be in ISO 8601 format (e.g. 2026-01-01T00:00:00Z). Leaderboard event windows are always aligned to the top of an hour in UTC.
